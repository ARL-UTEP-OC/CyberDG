Run


The Run screen allows for network setup, for which a static IP Address is assigned by default but can be configured to the user's convenience. There are three types of supported networks: Default Internal Network, NAT Network and Named Internal Network. Internal networks can be named by the user.

To run a scenario follow the instructions below:

(From the Landing Page select the Run button)

Once the Run button is selected you will view the network setup screen. This section includes the following options.



  • Network Setup Screen
    • Main Section Displays:
      • Machine Name
      • Network Name
      • IP Address



      • Edit:
        • Network Type:
          • This option allows the user to edit the type of network to use in that specific machine, the options are: NAT Network, Default Internal Network, Named Internal Network.

        • Machine IP:
          • This option allows the user to view and change the machines IP address. Although editing the machine's IP address should be done with great caution.

      • Cancel:
        • Allows the user to return to the network setup page without saving progress.
      • Save:
        • Allows the user to save the network changes made to the VM.

    • Back:
      • The Back button will take the user back to the configuration page.

    • Exit:
      • The Exit button on the top right corner will transport the user back to the Landing Page.

    • Next:
      • The Next button will take the user to the Build Screen.



  • Build Screen
    • This Section Displays:
      • Machine Name:
      • The name of the current machine that is displayed

      • VM OS:
      • The operating system the machine is currently running.

      • Files:
      • The files added to the machine.

      • Commands:
        • The commands (if any) that have been pre-determined by the user to run on the machine.

      • RDP to (Machine Name):
        • Allows the user to RDP into the machine that is currently selected.
        • *Note: Selecting this option directly opens the machine in a new tab allowing for the user to view and manipulate the VM as desired.*




      • Save as Base Machine:
        • Allows the user to save the current machine as a base machine for later use in other scenarios.
        • *Note: Selecting this option will take you to a screen to rename the machine for the use of a base machine*

    • Back:
      • The Back button will take the user back to the Network page.

    • Exit:
      • The Exit button on the top right corner will transport the user back to the Landing Page.

    • Run:
      • The Run button will take the user to the next screen entitled Running Page.



  • Running Screen
    • This Section Displays:
      • Machine Name:
        • The name of the current machine that is displayed

      • VM OS:
        • The operating system the machine is currently running.

      • Files:
        • The files added to the machine.

      • Collectors:
        • The collectors currently running on the VM(s).

      • RDP:
        • Allows the user to RDP into the machine that is currently selected.
        • *Note: Selecting this option directly opens the machine in a new tab allowing for the user to view and manipulate the VM as desired.*

      Second Row of Buttons Contain the Following Options:
      • View/Export Collector Logs:
        • *This option is currently Unavailable*

      • Re-Run Scenario:
        • This option allows the user to re-run the scenario multiple times.

        • Reconfigure Scenario:
          • *This option is currently Unavailable*

        Other Options On This Screen Include the Following:
      • Back:
        • The Back button will take the user back to the Build page.

      • Stop:
        • The Stop button will stop the VM(s) from running, allowing for the VM(s) to completely power off.

      • Exit:
        • The Exit button on the top right corner will allow you to exit the Running page and transport the user back to the Landing Page.
