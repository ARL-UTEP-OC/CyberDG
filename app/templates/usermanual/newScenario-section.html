New Scenario


Scenarios are the machine environments that users create in order to test exploits. Scenarios can contain up to one network of VMs that can be classified as either Exploits or Vulnerabilities. Scenarios can be identifies by a Scenario Name, an optional CVE Number and a short Description.

To create a New Scenario follow the instructions below:

(From the Landing Page select the New Scenario button)

Once the New Scenario button is selected you will be prompted to fill in the following options:




The image on the left shows the blank New Scenario modal. The image on the right shows a example of an acceptable input.

(*Note: 'Names can only contain letters, numbers, and hyphens.'*)


  • Scenario Name:
    • Enter the desired name of the scenario you wish to create.
    • *Note: Adding a name is required in order to create a new scenario.*

  • CVE Number: *This field is not required*
    • The CVE ID numbers are used by cybersecurity product/service vendors and researchers as a standard method for identifying vulnerabilities and for cross-linking with other repositories that also use CVE ID numbers.
    • The CVE ID numbers can be found on websites such as Exploit Database or the National Vulnerability Database to name a few.

  • Scenario Description:
    • This section allows for a small description of the scenario that will be displayed on the landing page underneath the selected scenario name from the available scenario list.
    • *Note: Adding a Description is required in order to create a new scenario *

  • Exploits:
    • This option allows a user to add an exploit file(s) that will run on the virtual machines.

    • To Add an Exploit File(s) to the Current Scenario:
      • If a file already exists in the system:
      • Select the file(s) from drop-down menu, select Save.

      • If a file does not already exist in the system and you would like to add it to the current scenario:
      • Select Add Files -> (See the Upload section for more information on how to upload a file.) -> Select Import ->(once all files have been added) Select Save.

  • Vulnerabilities:
    • This option allows a user to add an vulnerable file(s) that will run on the virtual machines.

    • To Add an Vulnerable File(s) to The Current Scenario:
      • If a file already exists in the system:
      • Select the file(s) from the drop-down menu, select Save.

      • If a file does not already exist in the system and you would like to add it to the current scenario:
      • Select Add Files -> (See the Upload section for more information on how to upload a file.) -> Select Import -> Select Save.

  • To finalize the scenario creation, select Save.

  • Once you have selected the Save button, the configuration section will open to take you through the next steps of creating a new scenario.
