Upload Files


Files that are to be used as Exploits or Vulnerabilities can be uploaded to the system in order to saved for future use, as well as prevent file duplication through hashing. Files are identified with a name, a description and an optional CVE Number.

To upload a file follow the instructions below:

(From the Landing Page Select the Upload Button)

Once the Upload button is selected you will be prompted to fill in the following options:




The image on the left shows the blank Upload modal. The image on the right shows a example of an acceptable input.

(*Note: 'Names can only contain letters, numbers, and hyphens.'*)


  • Upload Type:
    • The user will select what type of file they wish to upload, whether it is a Vulnerability file or an Exploit file.
    • *Note: Selecting an upload type is required in order to upload a new file.*

  • Upload Name:
    • Enter the desired name of the file you wish to upload.
    • *Note: Adding a name is required in order to upload a new file.*

  • File Description:
    • This section allows for a small description of the file that will be displayed on file window from the available files list.

  • CVE Number: *This field is not required*
    • The CVE ID numbers are used by cybersecurity product/service vendors and researchers as a standard method for identifying vulnerabilities and for cross-linking with other repositories that also use CVE ID numbers.
    • The CVE ID numbers can be found on websites such as Exploit Database or the National Vulnerability Database to name a few.

  • Browse:
    • Selecting this button allows the user to select the desired file they wish to upload from their computer's local directory or other external devices directory.

  • To finalize the file upload, select Upload.

  • Once you have selected the Upload button, the user will be transferred back to the Landing Page. The file will be uploaded to the system and will be displayed in the Files section.
