Configure


The Configure screen allows for further customization of scenarios, and the addition of VMs beyond those initially configured if desired. VMs can be customized in the following ways: Name, Type (Victim, Exploit, Victim/Exploit), OS, Exploits, Vulnerabilities, Base Memory, Processor, Network Type, Command Line Prompt and Command Line Order. Customized machines can be saved as Base VMs in order to be used in future scenarios.

To configure a machine follow the instructions below:

(From the Landing Page select a scenario from the Available Scenario list, then select the Configure button.)

Once the Configure section appears the following information will be displayed:



  • Main Section:
    • Displays the current machines available in the selected scenario:
      • Machine Name
      • Machine OS
      • Network Name
      • Files added to the machine
      • Command Line Prompts
      • Execution Order



  • Edit:
    • Allows the user to modify the machine configurations.
      • Machine Name:
        • Allows the user to change the name of the machine.
      • Type:
        • Allows the user to change the type of the machine (Victim, Attacker, Victim/Attacker).
      • VM OS:
        • Allows the user to change the operating system of the machine.
      • Exploits:
        • Allows the user to add more exploit files from the drop-down menu.
      • Vulnerabilities:
        • Allows the user to add more vulnerable files from the drop-down menu.
      • Add Files:
        • Allows the user to upload a file directly into a scenario.
      • Base Memory:
        • Allows the user to adjust the amount of memory allocated to the machine.
      • Processor:
        • Allows the user to adjust the processors allocated to the machine.
      • Network Type:
        • Allows the user to change the network type (Default Internal Network, Nat Network, or Named Internal Network) of the machine.
      • Command Line Prompt(s):
        • Allows the user to pass commands needed to run on the machine before the file is executed.
      • Command Line Order:
        • Allows the user to specify what commands need to execute in a specific order.
    • Back:
      • Allows the user to return to the configuration main page.
    • Save:
      • Allows the user to save the changes made to the VM.






  • Info:
    • Displays the information associated with the machine:
      • Database ID
      • Date of Creation
      • Machine Name
      • Machine IP
      • Machine RDP IP
      • Machine RDP Port
      • Machine Username
      • Machine Password
      • Machine Type
      • Machine Processors
      • Machine Memory
      • Machine Network Type
      • Machine Command Line Prompt(s)
      • Machine Command Line Execution Order
      • Machine Scenario ID
      • Machine OS ID
      • Machine Configuration
      • Machine File(s)
    • Cancel:
      • Allows the user to return to the configuration main page.
    • Delete Machine:
      • Allows the user to delete the machine.

  • Delete:
    • Allows the user to delete the machine.

    Once the user have completed the editing process, select Save to save the changes or Back to return back to the Configuration page.


Other options to configure a machine include:




  • Add Machine:

    • To add a machine, follow the instructions below:

    (From the Configure screen, click on Add Machine).

    Once the Add Machine button is selected the user will be prompted to fill in the following options:

    • Enter Machine Name:
      • Enter the desired name of the machine you wish to create.
      • *Note: Adding a name is required in order to add a new machine.*

    • Machine Type:
      • Select what type of machine you wish to create,(Victim,Attacker, Victim/Attacker).
      • *Note: Selecting a machine type is required in order to create a new machine.*

    • VM OS:
      • Select which operating system the machine will operate on.

    • Exploits:
      • Allows the user to add more exploit files from the dropdown menu.

    • Vulnerabilities:
      • Allows the user to add more vulnerable files from the dropdown menu.

    • Add Files:
      • Allows the user to upload a file directly into a scenario.

    • Base Memory:
      • Select the amount of RAM that is allocated to this machine. The base memory allocation can be between 4Mb and 16384Mb
      • The default amount of RAM will be set to 5000Mb

    • Processors:
      • Select the number of processors that can be allocated to this machine. The processor allocation can be between 1 and 4 processors
      • The default number of processors will be set to 2

    • Network Type:
      • Allows the user to change the network type from (Default Internal Network, Nat Network, or Named Internal Network).

    • Command Line Prompt(s):
      • Allows the user to pass commands needed to run on the machine before the file is executed on the machine.

    • Command Line Order:
      • Allows the user to specify what commands need to execute in a specific order.

    • To finalize the machine creation, select Save. Or select Back to cancel machine creation.

    • Once you have selected the Save button, the machine will be added/modified to the scenario.
  • Next:
    • To finalize the machine configuration, select Next.
    • Once you have selected the Next button, the Network Setup section will open to take you through the next steps of creating/editing a new scenario.

  • Exit:
    • The Exit button on the top right corner will transport the user back to the Landing Page.
